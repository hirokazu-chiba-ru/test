## 1. システム概要

## 3. アーキテクチャ

### 3.1 構成図

```
ブラウザ
  │
  │ HTTPS
  ▼
SageMaker プロキシ
/codeeditor/default/ports/5000/
  │
  │ HTTP
  ▼
Flask サーバー (0.0.0.0:5000)
  │
  ├── GET  /login         → login20260417.html を返す
  └── POST /auth          → JSON認証レスポンスを返す
```

### 3.2 認証フロー

```
1. ブラウザが /login にアクセス
2. Flask が login20260417.html を返す
3. ユーザーがユーザー名・パスワードを入力してログインボタンをクリック
4. JavaScript (fetch API) が /auth に POST (JSON)
5. Flask が認証情報を照合
   ├── 一致 → {"ok": true, "name": "<ユーザー名>"} を返す
   └── 不一致 → {"ok": false} + HTTP 401 を返す
6. JavaScript がレスポンスに応じてDOMを切り替え
   ├── 成功 → 成功画面を表示（ページ遷移なし）
   └── 失敗 → エラーメッセージを表示
```

### 3.3 SageMaker プロキシ対応

SageMaker Code Editor のポートプロキシには以下の制約がある：

| 制約 | 対応策 |
|---|---|
| フォームPOSTをインターセプトする | fetch API による非同期通信に変更 |
| ページ遷移時にリクエストを書き換える | 1ページSPA構成でページ遷移を廃止 |
| `username`/`password` パラメータを検知する | JSONフィールド名を `u`/`p` に変更 |

## 4. API仕様

### 4.1 GET /login

ログイン画面HTMLを返す。

| 項目 | 内容 |
|---|---|
| メソッド | GET |
| パス | `/login` `/codeeditor/default/ports/5000/login` |
| レスポンス | HTML (login20260417.html) |

### 4.2 POST /auth

認証処理を行いJSON形式で結果を返す。

| 項目 | 内容 |
|---|---|
| メソッド | POST |
| パス | `/auth` `/codeeditor/default/ports/5000/auth` |
| Content-Type | application/json |

**リクエストボディ:**
```json
{
  "u": "ユーザー名",
  "p": "パスワード"
}
```

**レスポンス（成功時）:**
```json
{
  "ok": true,
  "name": "admin"
}
```
HTTP ステータス: `200 OK`

**レスポンス（失敗時）:**
```json
{
  "ok": false
}
```
HTTP ステータス: `401 Unauthorized`
