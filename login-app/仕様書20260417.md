## 2. 画面仕様

### 2.1 画面構成

1ファイル（`login20260417.html`）にログイン画面と成功画面を統合した SPA（シングルページ）構成。
ページ遷移は行わず、JavaScript による DOM 切り替えで画面を制御する。

```
login20260417.html
├── #loginView   ... ログイン画面（初期表示）
└── #successView ... 成功画面（認証成功後に表示）
```

### 3.3 POST /auth

認証処理を行い結果を JSON で返す。

**リクエスト**

```
Content-Type: application/json
```

```json
{ "u": "ユーザー名", "p": "パスワード" }
```

| フィールド | 型 | 説明 |
|---|---|---|
| `u` | string | ユーザー名 |
| `p` | string | パスワード |

実装箇所: [`app20260417.py:24-26`](app20260417.py#L24-L26)
```python
data = request.get_json()
uid = data.get("u", "")
pwd = data.get("p", "")
```

**レスポンス（認証成功）**

HTTP ステータス: `200 OK`

```json
{ "ok": true, "name": "admin" }
```

**レスポンス（認証失敗）**

HTTP ステータス: `401 Unauthorized`

```json
{ "ok": false }
```

実装箇所: [`app20260417.py:27-29`](app20260417.py#L27-L29)
```python
if uid in USERS and USERS[uid] == pwd:
    return jsonify({"ok": True, "name": uid})
return jsonify({"ok": False}), 401
```

## 4. フロントエンド処理仕様

### 4.1 doLogin() 関数

認証ボタン押下時、または Enter キー押下時に実行される。

実装箇所: [`login20260417.html:117-153`](templates/login20260417.html#L117-L153)

```
1. uid・pwd を取得
2. いずれかが空なら処理中断
3. ボタンを非活性化、テキストを「認証中...」に変更
4. POST /auth に JSON で送信 (fetch API)
5. レスポンス処理:
   ├── ok=true → welcomeName にユーザー名を設定
   │           → #loginView を非表示
   │           → #successView を表示
   │           → body クラスを success-bg に変更
   ├── ok=false → #errorMsg を表示、ボタンを再活性化
   └── 例外発生 → エラーメッセージを「サーバーに接続できませんでした。」に変更して表示
```

### 4.2 doLogout() 関数

ログアウトボタン押下時に実行される。

実装箇所: [`login20260417.html:155-162`](templates/login20260417.html#L155-L162)

```
1. uid・pwd の入力値をクリア
2. #successView を非表示
3. #loginView を表示
4. #errorMsg を非表示
5. body クラスを login-bg に変更
```

### 4.3 Enterキーイベント

実装箇所: [`login20260417.html:165-167`](templates/login20260417.html#L165-L167)

```javascript
document.addEventListener('keydown', function(e) {
    if (e.key === 'Enter') doLogin();
});
```
